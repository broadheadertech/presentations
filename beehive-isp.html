Is it doable?

Straight answer, before the detail: yes — and the network half is genuinely boring, which is the highest compliment infrastructure can receive.

Proven

The network

MikroTik as a PPPoE server with a TP-Link dialling in from each room is the standard Philippine small-ISP build. Thousands of operators run it today. Nothing here is novel, and that is the point.

Scoped

The software

The loop that matters — pay, enable, lapse, suspend — is one table and two API calls. That ships in weeks. The full multi-property SaaS in the spec is a different project, and it should wait until this building has run for a season.

Not with TP-Link

Device-level control

"Device limit" and "connected devices" cannot work. A consumer TP-Link has no API and it NATs, so Beehive sees one session per room and never the phones behind it. Drop the feature, or change the room unit.

Everything else in the specification survives contact with this hardware. That one feature does not — better to know now than to promise it to a landlord.

The network

One MikroTik acts as the exchange. Everything below it is deliberately dumb.

How a room comes online

Five steps. The tenant sees none of them — they just plug in their router and it works, exactly like Converge at home.

01

Beehive creates the secret

A PPPoE username for the room — room-214 — pushed to the MikroTik with the plan's speed profile attached.

02

The TP-Link dials

WAN set to PPPoE, username and password entered once at install. It redials by itself forever after.

03

The gateway authenticates

Secret matches, session opens, the room gets an IP and lands in its speed profile. Beehive logs the session.

04

The room NATs

Tenant's phones and laptops sit behind their own router, on their own subnet, invisible to every other room.

05

PCQ shares what's left

One queue divides the uplink evenly across whoever is online. Ten rooms active, a tenth each. Forty active, a fortieth. No rules to write.

Because each room NATs, PCQ fair-shares per room, not per device. A tenant with five gadgets gets one share to split among their own — which is precisely the rule a landlord wants.

The whole product is one switch

Everything Beehive does reduces to flipping one field on one row, and letting the network react.

PPP SECRET · ROOM 214
name     room-214
profile  plan-25mbps
service  pppoe
disabled no  →  yes

Set disabled=yes, drop the live session, and room 214 is off. Set it back, the router redials within seconds. That is the entire enforcement mechanism.

Three people use this

The spec listed six roles. A single dorm needs three, and adding the others before they exist is how admin screens rot.

Role 01

Admin

Adds rooms and tenants, sets plans, overrides suspensions, sees the hive board and the audit log. In practice: the landlord, or you.

Role 02

Cashier

Takes cash, records it against a room, prints the receipt. Cannot change plans, cannot suspend, cannot edit tenants. Narrow on purpose — this is the role most likely to be a part-timer.

Role 03

Tenant

Sees their bill, pays it, gets a receipt. That's the whole surface. Every extra button here is a support call waiting to happen.

Technician, property owner, and super admin arrive when there's a second building. Not before.

What the tenant actually experiences

The measure of this system is that a tenant never has to learn it exists.

◔

Move-in: one sticker

Room number, PPPoE username, password. The installer sets it once on the TP-Link; the tenant never touches it.

◕

Their own WiFi name and password

They own the router. They set the SSID. They reboot it when they feel like it. Zero tickets about forgotten shared passwords.

◑

Payday: a link and a QR code

GCash or Maya from the phone. The room is back before they've closed the app.

◒

Move-out: nothing to collect

Disable the secret. The router stays on the wall for the next tenant. No hardware chased down a corridor.

The honest trade

They're behind two layers of NAT

Their router NATs, and the gateway NATs again. Most tenants will never notice. Some will: certain game consoles complain about strict NAT, and a few VPNs get awkward.

This is the price of dumb switches and zero VLANs, and it's usually worth paying. But say it out loud before a gamer signs a lease — not after.

Hardware, and what each box is for

Models come after the site walk. What matters now is that each layer has exactly one job.

The exchange

MikroTik gateway

The only box that needs skill

PPPoE server, PCQ, NAT, firewall. Configured once, at your desk. Size it for queued throughput, not the datasheet number — enabling queues bypasses fasttrack and the honest figure is a lot lower.

The wiring

TP-Link switches

Deliberately boring

One port per room. No VLANs, no config. If you buy managed anyway, turn on port isolation — it stops anyone unplugging a room's router and poking at the flat segment behind it.

The subscriber

TP-Link router, per room

A modem the tenant owns

WAN set to PPPoE, NATs for the room, radio serves that room only. Set the channel by hand at install — 20 MHz, fixed, 2.4 GHz off on most. Forty units left on auto will fight each other.

The concrete walls that make dorm WiFi hard are what make this work: each room's radio is isolated by the building itself.

How to build it without disappearing for a year

Each phase leaves you with something that runs. The building earns money from the end of phase two.

01

The network

Weeks 1–3

Fiber, rack, drops, forty routers, PPPoE server, PCQ. No software at all. Run it on a spreadsheet and WinBox.

02

The loop

Weeks 4–8

Rooms, tenants, plans, invoices, payments, enable/disable via the API. The hive board. This is Beehive v1 and it's enough to stop using the spreadsheet.

03

Self-service

Weeks 9–12

Tenant login, online payment, automatic reactivation, receipts. The point where the cashier stops being a bottleneck.

04

Building two

After a season

Only now: multi-property, more roles, helpdesk, reports. By then you'll know which of them anyone actually asked for.

Phase 1 has no software in it on purpose. Two months of running forty rooms by hand will tell you more about what Beehive should be than any amount of specification.
